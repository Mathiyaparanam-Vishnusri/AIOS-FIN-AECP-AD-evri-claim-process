# AIOS GPT Brain Production Skill Pack — START HERE

## Purpose

This skill pack is designed for a dedicated GPT project that acts as the thinking layer for staff task execution.

GPT is the brain. Claude Code is the worker. GitHub, PostgreSQL, workflow exports, evidence files, and validation logs are the proof layer.

The purpose is to stop uncontrolled work, duplicated assets, invisible progress, weak evidence, and drift from the original objective.

## How To Use

1. Create the dedicated GPT project.  
2. Paste the full Dedicated GPT Project Instruction into the project instruction setting.  
3. Upload all files in this skill pack into the GPT project Sources.  
4. Use GPT to create every Claude Code prompt.  
5. Paste every Claude Code output back into GPT.  
6. Ask GPT to review the output before the next step.  
7. Save evidence in the approved local/VM folder and GitHub.  
8. Run the drift prevention prompt repeatedly through the day.

## Core Operating Rule

No direct planning inside Claude Code.

The user should not type free-form task instructions into Claude Code.

Claude Code should mainly receive GPT-generated prompts.

Direct commands allowed in Claude Code should normally be limited to:

- commit  
- push  
- shutdown  
- simple confirmation commands required by the tool

Everything else should be structured by GPT.

## Required Daily Outcome

Every working day must create or improve at least one reusable, evidence-backed, queryable asset.

Examples:

- reusable prompt  
- evidence pack  
- validation report  
- PostgreSQL object or metadata proposal  
- workflow export  
- GitHub documentation  
- capability file  
- decision rule  
- duplicate-risk report  
- closure handover note

## Non-Negotiable Standard

No evidence \= no completed work. No reusable asset \= weak work. No existing asset search \= no approval to create new work. No GPT review of Claude output \= uncontrolled execution. No unknown developer readiness \= incomplete handover.

# Skill 10 — Unknown Developer Test

## Purpose

This skill ensures continuity.

A new unknown developer should be able to continue the work safely without verbal explanation.

## Core Rule

Every important asset, task, or handover must pass the Unknown Developer Test.

## Test Scenario

Assume a developer who has never seen this project opens the repository tomorrow.

Can they understand:

- what the work is
- why it exists
- what was completed
- what remains unfinished
- where the evidence is
- what should happen next
- what must not be touched
- what risks exist
- what validation is required

If not, the task is incomplete.

## Required Handover Fields

Every closure or handover file should include:

- objective
- current status
- files created or changed
- evidence location
- Git commit/path
- systems touched
- systems not touched
- known risks
- duplicate-risk status
- validation status
- next action
- pass/fail status

## Unknown Developer Failure Signals

- “Ask me tomorrow”
- “I know where it is”
- “Claude knows”
- “It is in the chat”
- “It is on my desktop”
- “I will explain”
- “The file name is obvious”
- “We discussed this already”

These fail the test.

## Pass / Fail Rule

PASS if an unknown developer can continue from saved files and evidence only.

FAIL if continuation requires verbal explanation, memory, chat history, or creator assistance.

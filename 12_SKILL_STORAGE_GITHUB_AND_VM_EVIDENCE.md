# Skill 12 — Storage, VM, GitHub, and Evidence Discipline

## Purpose

This skill ensures work is preserved outside chat.

Chat is not storage.
Memory is not storage.
Claude output is not evidence unless saved.

## Core Rule

Important outputs must be saved in an approved working location and pushed or referenced through GitHub where appropriate.

Preferred storage order:

1. Dedicated VM working folder
2. Dedicated project working folder
3. Approved repository location
4. GitHub commit
5. Evidence register or closure file

## What Must Be Saved

- Claude discovery output
- Claude validation output
- created files
- modified files
- evidence packs
- workflow exports
- SQL outputs
- closure handover
- prompt used for important execution
- final GPT review

## Folder Discipline

Use clear folder names.

Avoid:

- test
- final
- new
- temp
- random
- notes
- old

Prefer:

- evidence
- validation
- workflows
- handover
- prompts
- capability
- sql
- closure

## GitHub Rule

If work is important, it should have a GitHub path or commit.

A local-only file is not considered fully complete unless explicitly approved as temporary staging.

## VM Rule

When work is executed on a VM, outputs should be saved into the dedicated project folder on the VM.

Do not leave important files scattered across Downloads, Desktop, or temporary locations.

## Evidence Link Rule

Every important asset should include a link/path to evidence.

Every evidence file should state which asset it supports.

## Pass / Fail Rule

PASS if important work is saved in a dedicated folder and GitHub/evidence path is recorded.

FAIL if important work remains only in chat, Claude output, local temp folders, or memory.

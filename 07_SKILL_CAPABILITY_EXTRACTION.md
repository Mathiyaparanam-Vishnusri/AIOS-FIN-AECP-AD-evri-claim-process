# Skill 07 — Capability Extraction

## Purpose

This skill ensures solved problems become reusable company capability.

The organization should not solve the same problem repeatedly.

Every useful discovery should be evaluated for reuse.

## Core Rule

Problem fixed is not enough.

The correct closure is:

Problem fixed
→ method extracted
→ capability captured
→ evidence linked
→ reuse path defined

## What Is A Capability?

A capability is a repeatable method the organization can use again.

Examples:

- duplicate detection workflow
- evidence validation checklist
- prompt pattern
- PostgreSQL inspection method
- n8n workflow review method
- customer complaint classification rule
- PPC leakage detection rule
- stock recovery prioritization rule
- unknown developer handover pattern
- closure checklist
- validation query pack

## Capability Extraction Questions

After any task, GPT must ask:

1. Did we discover a repeatable method?
2. Did we discover a better prompt?
3. Did we discover a validation rule?
4. Did we discover a decision rule?
5. Did we discover a workflow pattern?
6. Did we discover a common failure mode?
7. Did we discover something another staff member can reuse?
8. Should this be saved as a capability file?

## Capability File Minimum Fields

A capability file should include:

- capability name
- purpose
- business problem solved
- when to use
- when not to use
- required inputs
- execution steps
- evidence required
- pass/fail rule
- owner/reviewer
- known limits
- reuse path

## Capability Promotion Levels

| Level | Meaning |
|---|---|
| DRAFT | identified but not validated |
| TESTED | used once with evidence |
| REUSABLE | can be used by others |
| GOVERNED | approved as standard |
| ACTIVE | used in cadence |

## Failure Modes

| Failure | Damage |
|---|---|
| Problem fixed but method not captured | Same issue repeats |
| Useful prompt not saved | Future prompt quality drops |
| Validation method not documented | Future work less reliable |
| Capability created without evidence | Weak reusable asset |
| Capability not queryable | GPT cannot reuse it |

## Pass / Fail Rule

PASS if repeatable learning is captured when found.

FAIL if useful methods remain only in chat, memory, or verbal explanation.

# Skill 02 — GPT → Claude → GPT Execution Loop

## Purpose

This skill defines the controlled execution loop.

The loop prevents drift, duplication, undocumented work, and uncontrolled coding.

## Standard Loop

1. Requirement received
2. GPT clarifies objective
3. GPT identifies business question
4. GPT checks existing asset requirements
5. GPT creates Claude Code discovery prompt
6. Claude Code executes discovery
7. User pastes Claude output into GPT
8. GPT reviews output
9. GPT creates implementation or correction prompt
10. Claude executes
11. Output returns to GPT
12. GPT reviews evidence
13. GPT creates closure prompt
14. Claude creates closure asset
15. GPT confirms completion status

## Why This Loop Exists

Claude Code can execute quickly.
Fast execution without thinking creates duplicate files, wrong assumptions, and hidden logic.

GPT must slow the process only enough to protect ROI, evidence, queryability, and reusability.

## Required Claude Prompt Structure

Every Claude prompt generated by GPT should include:

- Role
- Objective
- Context
- Scope
- Existing asset search requirement
- Systems to inspect
- Systems not to touch
- Evidence required
- Output format
- Pass/fail rule
- Stop conditions

## Required Output From Claude

Claude must return:

- what it checked
- what it found
- files touched
- assets created
- evidence created
- duplicate risks
- unresolved gaps
- recommended next step
- final decision: GREEN / AMBER / RED

## Stop Conditions

Claude must stop if:

- source of truth is unclear
- duplicate risk is detected
- production changes are required
- business logic is unclear
- evidence cannot be generated
- requirement contradicts existing asset
- task needs human validation

## GPT Review Decision

After reviewing Claude output, GPT must decide:

GREEN — continue or close.
AMBER — fix gaps before continuing.
RED — stop and redesign.

## Bad Loop Example

User: “Claude, create a new n8n workflow for this.”
Claude: creates workflow.
No asset search.
No duplicate check.
No evidence.
No GitHub path.
No GPT review.

This fails.

## Good Loop Example

User asks GPT for help.
GPT creates discovery prompt:
“Search existing n8n workflows, GitHub exports, documentation, and capability files before recommending creation.”
Claude discovers an existing partial workflow.
GPT decides to extend instead of create new.
Claude updates documentation and exports evidence.
GPT reviews and closes.

This passes.

## Pass / Fail Rule

PASS if every major execution cycle includes GPT prompt, Claude output, GPT review, evidence, and next decision.

FAIL if Claude output is accepted without GPT review.
